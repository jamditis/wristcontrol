WristControl Documentation

WristControl transforms your Samsung Galaxy Watch into a powerful input device, enabling hands-free computer control through natural gestures and voice commands.

Quick Start

Get up and running in under 5 minutes. Install the desktop app, connect your watch, and start controlling.

System Requirements

Desktop Computer

  • Windows 10/11 (64-bit) with Bluetooth LE support
  • macOS 11+ (Big Sur or later)
  • Linux with BlueZ 5.50+ and PulseAudio/PipeWire
  • Python 3.10 or later (for source installation)
  • Bluetooth 4.0+ adapter with BLE support

Smartwatch

  • Samsung Galaxy Watch 4 or newer (Wear OS)
  • Older Galaxy Watch models (Tizen) - limited support
  • Wear OS 3.0 or later recommended
Note: Other Wear OS watches may work but are not officially tested.

Installation

Option 1: Pre-built Binaries (Recommended)

Download the latest release for your platform:

Windows wristcontrol-win64.exe macOS WristControl.dmg Linux wristcontrol.AppImage

Option 2: Install from Source

Terminal 
# Clone the repository
git clone https://github.com/wristcontrol/wristcontrol.git
cd wristcontrol/desktop

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -e ".[dev]"

# Run the application
wristcontrol

Option 3: pip Install

Terminal 
pip install wristcontrol
wristcontrol

Install the WristControl companion app on your Galaxy Watch from the Play Store.

4

Launch both apps and follow the pairing wizard. Adjust sensitivity to your preference.

Connecting Your Watch

WristControl uses Bluetooth Low Energy (BLE) for efficient, low-latency communication with your watch.

Pairing Process

  1. Open WristControl on your computer - it will start scanning for devices
  2. Open the WristControl app on your Galaxy Watch
  3. Select your watch from the list of discovered devices
  4. Accept the pairing request on your watch
  5. Wait for the connection to establish (status indicator turns green)
Important: Keep your watch within 10 meters of your computer for reliable connection. Walls and other obstacles may reduce range.

Cursor Control

WristControl offers two cursor control modes:

Relative Mode (Default)

Tilting your hand moves the cursor in that direction. The speed of movement depends on the degree of tilt. Return your hand to neutral to stop cursor movement.

Absolute Mode

Your hand position maps directly to screen position. Point at where you want the cursor to go. Best for users comfortable with pointer-based control.

Tip: Start with Relative mode and low sensitivity, then increase as you become comfortable.

Settings Overview

Configuration is stored in ~/.config/wristcontrol/config.json and can be edited through the app or directly.

config.json 
{
  "cursor": {
    "sensitivity": 1.0,
    "dead_zone": 0.1,
    "acceleration": 1.5,
    "smoothing": 0.3,
    "mode": "relative"
  },
  "gestures": {
    "tap_threshold": 0.5,
    "double_tap_window": 0.3,
    "hold_duration": 0.5,
    "enabled_gestures": ["tap", "double_tap", "hold", "wrist_flick"]
  },
  "voice": {
    "enabled": true,
    "backend": "whisper_local",
    "activation_mode": "push_to_talk",
    "wake_word": "computer"
  },
  "bluetooth": {
    "device_name": "Galaxy Watch",
    "auto_reconnect": true
  }
}

Sensitivity Tuning

Cursor Sensitivity

  • sensitivity (0.1 - 3.0): Overall cursor speed multiplier
  • dead_zone (0.0 - 0.5): Movement threshold to filter noise
  • acceleration (1.0 - 3.0): Speed increase for larger movements
  • smoothing (0.0 - 0.9): Movement averaging for stability

Gesture Sensitivity

  • tap_threshold: Force required to register a tap
  • double_tap_window: Max time between taps (seconds)
  • hold_duration: Time to register a hold gesture

Custom Commands

You can define custom voice commands that execute specific actions or keyboard shortcuts.

Python API
from wristcontrol.voice import VoiceProcessor

voice = VoiceProcessor(config)

# Add custom command
voice.register_command(
    "launch browser",
    lambda: os.system("open -a Safari")  # macOS example
)

# Add command with parameters
voice.register_command(
    "search for *",
    lambda query: webbrowser.open(f"https://google.com/search?q={query}")
)

API Reference

WristControl exposes a Python API for integration and extension.

Core Classes

WristControlApp
from wristcontrol.core import WristControlApp

app = WristControlApp()
await app.start()

# Access components
app.bluetooth_manager  # BluetoothManager instance
app.gesture_processor  # GestureProcessor instance
app.voice_processor    # VoiceProcessor instance
app.cursor_controller  # CursorController instance

await app.stop()

Event Callbacks

Event Handling
# Register gesture callback
def on_gesture(event):
    print(f"Gesture: {event.gesture_type}, confidence: {event.confidence}")

gesture_processor.on_gesture = on_gesture

# Register sensor data callback
def on_sensors(data):
    print(f"Acceleration: {data.acceleration}")
    print(f"Gyroscope: {data.gyroscope}")

bluetooth_manager.on_sensor_data = on_sensors

Troubleshooting

Connection Issues

Watch not appearing in device list
  • Ensure Bluetooth is enabled on both devices
  • Make sure the watch is not connected to another device
  • Restart Bluetooth on your computer
  • On Linux, ensure BlueZ is properly configured
Frequent disconnections
  • Move closer to your computer (within 3 meters)
  • Remove interference from other Bluetooth devices
  • Check watch battery level (low battery affects BLE)
  • Update Bluetooth drivers on your computer

Gesture Recognition

Gestures not being detected
  • Ensure the watch is worn snugly on your wrist
  • Check that gesture recognition is enabled in settings
  • Try recalibrating the gesture sensor
  • Increase tap_threshold if getting false positives
Too many accidental clicks
  • Increase the dead_zone value in cursor settings
  • Increase tap_threshold in gesture settings
  • Enable palm-up detection to pause control when needed

Frequently Asked Questions

Does WristControl work with Apple Watch?

Currently, WristControl only supports Samsung Galaxy Watch and other Wear OS devices. Apple Watch support is planned for a future release.

What is the battery impact on my watch?

WristControl is designed for efficiency. Typical usage consumes about 15-20% battery per hour of active use. Battery life is estimated at 4+ hours of continuous use.

Can I use WristControl for gaming?

WristControl works best for productivity tasks. While basic gaming is possible, the latency (~50ms) may not be suitable for fast-paced or competitive games.

Is voice data sent to the cloud?

By default, WristControl uses local Whisper processing - all voice recognition happens on your computer. Cloud options are available but opt-in only.

Can I use WristControl while walking?

Yes! WristControl maintains 90%+ accuracy while walking. Motion compensation algorithms filter out walking movements from intentional gestures.
